Hexo-Next博客搭建与使用

发表于 2023-02-01 更新于 2023-02-06 阅读次数：

出于巩固知识温故知新的目的，以及梳理一下长期以来的个人笔记，决定重新翻新一下博客。由于本人不熟悉前端开发所以先找一种简单容易上手框架。折腾了一番最终选择了GitHub Pages + Hexo + Next。为了方便以后折腾和帮助有同样需求的人，在此记录一下本博客的搭建方法。并且在以后的使用和学习中发现新的知识点也会持续对此笔记进行修正和补充。

准备工作

  1. Github
    先提醒一下Github Pages官方文档里有几点建议和限制 ，大家看是否能满足个人需求再决定是否使用。使用username.github.io为仓库名创建一个Public仓库，username为GitHub用户名。
  2. Git
    前往Git官网 寻找合适自己的版本安装。
  3. NodeJs
    前往NodeJs官网 下载适合自己的版本安装。
    NodeJs默认的全局包路径在C盘，想要修改的可以修改一下。先执行命令 再修改环境变量。

部署Hexo

建议使用管理员权限执行命令

  1. 安装Hexo
  2. 创建项目并初始化
  3. 生成静态文件并启动本地服务器 这时通过提示访问 http://localhost:4000 就可以在本地看到一个初始化的博客。hexo -h 也可以看到命令各种选项的说明。
  4. 使用Hexo部署到Github
    本地测试成功后，进行部署。
  • 先修改博客根目录配置文件/_config.yml 根据自己的git情况配置
  • 执行部署命令 通过账户验证后即可在Github的仓库中看到刚刚初始化的网站静态文件。
  • 设置Github Pages
    前往 Settings -> Pages，将 Source 选择为 Deploy form a branch，在 Branch 选择分支，耐心等待一会刷新该页面会在上方看到 Your site is live at https://username.github.io/ 即表示部署成功，访问域名即可。

    等待过程很慢，可以稍微多等一会，大概20分钟如果还没部署成功可以尝试依次折腾一下

    1. 看看仓库是否为Public
    2. 看看仓库名称是否打错，一定要是username.github.io
    3. 看看仓库中根目录是否有index.html文件，如果没有则去hexo重新生成部署
    4. 如果找不出任何异常就试试在Pages设置中将Branch选择None，点一下Save，再选回来，点一下Save。

不使用Hexo直接用Git将项目推送到GitHub的项目仓库中也可以。

浏览统计

NexT有三种统计插件分别对应设置名称leancloud_visitors，firestore，busuanzi_count。前两种需要进行注册相应的账号，第三种直接打开配置即可。busuanzi在博客本地预览时数据可能不正确，不要担心部署后就正常了。

文章写作

  • 创建一篇新的文章
    layout代表想要使用的布局，不加时使用默认布局，在_config.yml当中default_layout 项设置，默认是post布局。 使用post布局时文件被储存在blog_root/source/_post。而后使用Markdown语法书写自己的内容即可。
  • 使用草稿
    创建文章时使用draft布局时代表创建一个草稿，草稿文件被储存在blog_root/source/_drafts。这类文章被称为草稿。草稿默认不被显示在页面种，除非在预览时使用--draft选项，或在_config.yml当中将render_drafts设置为true。
    当你想要发布草稿时使用
  • Front-matter
    Front-matter 是文件最上方以 — 分隔的区域，用于指定个别文件的变量。前面提过的tags和categories就是在这里指定。
  • 主页文章预览
    在文章中加入<!-- more -->标签，以指定首页显示文章预览的内容。
